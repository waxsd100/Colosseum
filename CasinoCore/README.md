# CasinoCore

Minecraft サーバー上でカジノイベントを運営するためのプラグインです。
プレイヤーは所持金（Economy）でチップ（カーペットアイテム）を購入し、テーブルゲームを楽しめます。

---

## 概要

- カジノモードの ON/OFF 管理（全体 or 個別プレイヤー）
- カジノモード中のプレイヤーを自動でアドベンチャーモードに切替、カジノシザース（バインドカース付きハサミ）を配布
- カジノ終了時に全プレイヤーのチップを自動回収・換金し、損益レポートを送信
- プレイヤー統計（セッション参加回数・勝敗・最大利益/損失など）
- ランキングは ChipLib の RankingManager に統合（`casino` カテゴリとして記録）
- データの非同期保存によるサーバー負荷軽減

## 依存関係

| 種別      | 名前                                                             | 用途                            |
|---------|----------------------------------------------------------------|-------------------------------|
| 必須プラグイン | [Vault](https://github.com/MilkBowl/Vault)                     | 経済（Economy）API                |
| 必須プラグイン | **ChipLib**                                                    | チップ管理・ランキング                   |
| 必須プラグイン | Vault 対応経済プラグイン                                                | 所持金の管理                        |
| ライブラリ   | [BindingCurseLib](https://github.com/waxsd100/BindingCurseLib) | カジノシザースの所有者管理（Shadow JAR に同梱） |

依存関係チェーン:

```
ChipLib（プラグイン） ← CasinoCore ← ArenaCore
```

## コマンド一覧

### 管理者コマンド `/casino`

| コマンド                    | 説明                              | 権限             |
|-------------------------|---------------------------------|----------------|
| `/casino on`            | カジノモードを開始（全プレイヤーをアドベンチャーモードに変更） | `casino.admin` |
| `/casino on <プレイヤー>`    | 個別プレイヤーをカジノモードに追加               | `casino.admin` |
| `/casino off`           | カジノモードを終了（全員のチップを換金・ゲームモード復元）   | `casino.admin` |
| `/casino off <プレイヤー>`   | 個別プレイヤーをカジノモードから退出（チップ換金・モード復元） | `casino.admin` |
| `/casino status`        | カジノモードの ON/OFF 状態を表示            | なし             |
| `/casino ranking`       | カジノ累計損益ランキングを表示                 | なし             |
| `/casino ranking reset` | ランキングと全プレイヤー統計をリセット             | `casino.admin` |
| `/casino stats [プレイヤー]` | プレイヤーの詳細統計を表示（省略時は自分）           | なし             |

#### 使用例

```
# カジノモードを開始
/casino on

# 特定プレイヤーだけカジノに追加
/casino on Steve

# カジノモードを終了（全員換金）
/casino off

# 特定プレイヤーだけ退出
/casino off Steve

# 現在の状態を確認
/casino status

# ランキング表示
/casino ranking

# Steve の統計を表示
/casino stats Steve
```

### プレイヤーコマンド

チップの購入・情報・換金は **ChipLib** が提供する `/chip` コマンドを使用します。
ランキング表示は `/ranking` コマンド（ChipLib 提供）で行えます。

## 権限（パーミッション）

| パーミッション        | 説明                          | デフォルト |
|----------------|-----------------------------|-------|
| `casino.admin` | `/casino` コマンドの管理操作（on/off） | OP    |

> `/casino status`, `/casino ranking`, `/casino stats` は権限なしで使用可能です。

## 設定ファイル

`plugins/CasinoCore/config.yml`

```yaml
# チップの最大購入額（自動分割モード時, 0 = 無制限）
max-buy: 1000000

# ランキング表示件数（1〜100, 推奨: 10）
ranking-size: 10

# チップの最小購入額
min-buy: 100

# チップの換金手数料率 (0.0 〜 1.0, 推奨: 0.0〜0.05)
cashout-fee: 0.0

# デバッグログ出力 (true/false)
debug: false
```

| 設定キー           | 説明                            | デフォルト値    |
|----------------|-------------------------------|-----------|
| `max-buy`      | 一度に購入できるチップの最大合計額（0 = 無制限）    | `1000000` |
| `ranking-size` | `/casino ranking` で表示するプレイヤー数 | `10`      |
| `min-buy`      | チップの最小購入額                     | `100`     |
| `cashout-fee`  | チップ換金時の手数料率（0.0 で手数料なし）       | `0.0`     |
| `debug`        | デバッグログの出力切替                   | `false`   |

### 自動保存されるデータ

`plugins/CasinoCore/data.yml` に以下が自動保存されます（手動編集は非推奨）:

- **ランタイム状態**: カジノ参加中プレイヤー、セッション購入記録、保存済みゲームモード
- **プレイヤー統計**: セッション参加回数、累計購入/換金額、勝敗、初回/最終参加日時

> **📝 ランキングデータについて**
> ランキングデータは ChipLib の `ranking_data.yml` に統合されています。`/ranking casino` で表示できます。

## カジノの運営フロー

```
┌─────────────────────────────────────────────────────┐
│ /casino on                                           │
│   ├─ ゲームモード保存 → アドベンチャーモードに変更    │
│   ├─ カジノシザース配布（バインドカース付き）         │
│   ├─ keepInventory を ON に設定                      │
│   ├─ ChipLib にチップ購入を許可                      │
│   └─ カジノ開始を全体通知                            │
├─────────────────────────────────────────────────────┤
│ プレイヤーがゲームを楽しむ                            │
│   ├─ /chip <金額> でチップ購入                        │
│   ├─ カーペットを設置して賭ける                       │
│   ├─ /chip balance で残高確認                        │
│   └─ /chip cashout で途中換金も可                    │
├─────────────────────────────────────────────────────┤
│ /casino off                                          │
│   ├─ 全プレイヤーのチップを自動回収・換金             │
│   ├─ 損益レポート送信                                │
│   ├─ ゲームモード復元                                │
│   ├─ カジノシザース回収                              │
│   ├─ keepInventory を元の値に復元                    │
│   ├─ ChipLib のチップ購入許可を解除                   │
│   └─ ランキング・統計を非同期保存                    │
└─────────────────────────────────────────────────────┘
```

## FAQ・トラブルシューティング

### Q: カジノ中にサーバーがクラッシュしたら？

`data.yml` に参加プレイヤーとゲームモードが保存されているため、再起動後に `/casino off` で正常に復旧できます。

### Q: チップを購入できない

- カジノモードが ON になっているか確認してください
- Vault 対応の経済プラグインが導入されているか確認してください
- 所持金が足りているか確認してください
- インベントリに空きスロットがあるか確認してください

### Q: カジノシザースを落としてしまった

カジノシザースにはバインドカースが付与されているため、通常は落とせません。万が一の場合は `/casino off` → `/casino on`
で再配布されます。

### Q: ランキングをリセットしたい

`/casino ranking reset` を実行するか、`/ranking reset casino` を実行してください。

## ビルド方法

プロジェクトルートで以下のコマンドを実行します:

```bash
# CasinoCore のみビルド
./gradlew :CasinoCore:build

# CasinoCore のテスト実行
./gradlew :CasinoCore:test

# テストサーバーで実行（Paper）
./gradlew :CasinoCore:runServer
```

ビルド成果物: `CasinoCore/build/libs/CasinoCore-3.0.0.jar`

> **📝 備考**
> - Shadow JAR プラグインにより、ChipLib（NBT-API 含む）と BindingCurseLib が同梱されます。
> - Java 17 以上が必要です。
