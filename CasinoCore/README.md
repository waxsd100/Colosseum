# CasinoCore

Minecraft サーバー上でカジノイベントを運営するためのプラグインです。  
プレイヤーは所持金（Economy）でチップ（カーペットアイテム）を購入し、テーブルゲームを楽しめます。

## 導入方法

### 前提条件

| 項目 | バージョン |
|------|----------|
| Minecraft | 1.20.1 以上 |
| Java | 17 以上 |
| [Vault](https://github.com/MilkBowl/Vault) | 必須 |
| Vault対応経済プラグイン | 必須 (例: EmeraldBank) |

### インストール

1. `CasinoCore-3.0.0.jar` を `plugins/` フォルダに配置
2. サーバーを起動（`plugins/CasinoCore/config.yml` が自動生成されます）
3. 必要に応じて `config.yml` を編集し、`/reload confirm` で反映

## クイックスタート（5分で始めるカジノ）

### Step 1 — カジノを開始する

```
/casino on
```

サーバー全体にカジノ開始が通知されます。カジノエリアにいるプレイヤーは自動的に以下の処理が行われます：

- ゲームモードが **アドベンチャーモード** に変更
- カーペット回収用の **カジノシザース**（バインドカース付きハサミ）を配布
- 元のゲームモードを内部で保存

### Step 2 — プレイヤーがチップを購入する

```
/chip 10000
```

所持金 10,000 E 分のチップが最適な組み合わせで自動分割され、インベントリに配布されます。

```
/chip 100 5
```

100 E チップを 5 枚購入します（合計 500 E）。

### Step 3 — ゲームで遊ぶ

プレイヤーはアドベンチャーモードでカーペット（チップ）を設置可能なブロックの上に置いて遊びます。  
カジノシザースでカーペットを壊すと、手元にチップが戻ります。

### Step 4 — カジノを終了する

```
/casino off
```

以下が自動で行われます：

- 全プレイヤーの手持ちチップを回収・換金
- 損益レポートの送信（購入額 / 換金額 / 損益 / 売却内訳）
- ゲームモードを元に戻す
- カジノシザースを回収
- ランキング・統計データを非同期保存

## コマンド一覧

### 管理者コマンド `/casino`

| コマンド | 説明 | 権限 |
|---------|------|------|
| `/casino on` | カジノを開始 | `casino.admin` |
| `/casino on <プレイヤー>` | 個別プレイヤーをカジノに追加 | `casino.admin` |
| `/casino off` | カジノを終了（全員換金） | `casino.admin` |
| `/casino off <プレイヤー>` | 個別プレイヤーを退出 | `casino.admin` |
| `/casino status` | カジノの動作状態を確認 | なし |
| `/casino ranking` | 累計損益ランキングを表示 | なし |
| `/casino ranking reset` | ランキング・統計をリセット | `casino.admin` |
| `/casino stats [プレイヤー]` | プレイヤー統計を表示 | なし |

### プレイヤーコマンド `/chip`

| コマンド | 説明 | 権限 |
|---------|------|------|
| `/chip <金額>` | 金額分のチップを最少枚数で購入 | `chip.use` |
| `/chip <額面> <枚数>` | 指定チップを指定枚数購入 | `chip.use` |
| `/chip info` | チップの額面・色対応表を表示 | `chip.use` |
| `/chip balance` | 手持ちチップの内訳と合計額を表示 | `chip.use` |
| `/chip cashout` | 手持ちチップを全額換金 | `chip.use` |

## 権限（パーミッション）

| パーミッション | 説明 | デフォルト |
|--------------|------|----------|
| `casino.admin` | `/casino` コマンドの使用 | OP |
| `chip.use` | `/chip` コマンドの使用 | 全員 |

## 設定ファイル

`plugins/CasinoCore/config.yml`

```yaml
# 一度の /chip コマンドで購入できるチップの最大合計額
max-buy: 1000000

# /casino ranking で表示するプレイヤー数
ranking-size: 10
```

### 自動保存されるデータ

`plugins/CasinoCore/data.yml` に以下が保存されます（手動編集は非推奨）：

- **ランタイム状態**: カジノ参加中プレイヤー、セッション購入記録、保存済みゲームモード
- **ランキング**: 全プレイヤーの累計損益
- **プレイヤー統計**: セッション参加回数、初回/最終参加日時、最大利益/損失

```
┌─────────────────────────────────────────────────┐
│ /casino on                                       │
│   ├─ ゲームモード保存 → アドベンチャーモードに変更  │
│   ├─ カジノシザース配布（バインドカース付き）       │
│   └─ カジノ開始を全体通知                         │
├─────────────────────────────────────────────────┤
│ プレイヤーがゲームを楽しむ                         │
│   ├─ /chip <金額> でチップ購入                    │
│   ├─ カーペットを設置して賭ける                    │
│   ├─ /chip balance で残高確認                     │
│   └─ /chip cashout で途中換金も可                 │
├─────────────────────────────────────────────────┤
│ /casino off                                      │
│   ├─ 全プレイヤーのチップを自動回収・換金           │
│   ├─ 損益レポート送信                             │
│   ├─ ゲームモード復元                             │
│   ├─ カジノシザース回収                           │
│   └─ ランキング・統計を非同期保存                  │
└─────────────────────────────────────────────────┘
```

| 黄色 | YELLOW_CARPET | 5,000 E |
| 橙色 | ORANGE_CARPET | 10,000 E |
| 桃色 | PINK_CARPET | 50,000 E |
| 赤色 | RED_CARPET | 100,000 E |
| 白色 | WHITE_CARPET | 500,000 E |
| 黒色 | BLACK_CARPET | 1,000,000 E |

### 自動分割の仕組み

`/chip 12345` と入力すると、貪欲法（大きい額面から順に）で最少枚数に分割されます：

| チップ | 枚数 |
|-------|------|
| 10,000 E (橙色) | 1 枚 |
| 1,000 E (黄緑色) | 2 枚 |
| 100 E (浅葱色) | 3 枚 |
| 10 E (青色) | 4 枚 |
| 5 E (紫色) | 1 枚 |
| **合計** | **11 枚** |

### 換金時のレポート

`/casino off` または `/chip cashout` 時に、以下の情報が表示されます：

- 購入額 / 換金額 / 損益（+/−/±）
- 勝ち・負け・引き分けの判定
- 全13種チップの売却内訳

## FAQ・トラブルシューティング

### Q: カジノ中にサーバーがクラッシュしたら？

`data.yml` に参加プレイヤーとゲームモードが保存されているため、再起動後に `/casino off` で正常に復旧できます。

### Q: チップを購入できない

- Vault 対応の経済プラグインが導入されているか確認してください
- 所持金が足りているか確認してください
- インベントリに空きスロットがあるか確認してください
- `max-buy` の上限を超えていないか確認してください

### Q: カジノシザースを落としてしまった

カジノシザースにはバインドカースが付与されているため、通常は落とせません。万が一の場合は `/casino off` → `/casino on` で再配布されます。

### Q: ランキングをリセットしたい

`plugins/CasinoCore/data.yml` の `ranking` セクションを削除してサーバーを再起動してください。
